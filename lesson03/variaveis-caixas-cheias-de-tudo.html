Dinâmico não é genérico

Para quem nunca programou numa linguagem com tipagem dinâmica, pode demorar um pouco até considerar algo agradável (ou até correto) de se fazer. Em Python, o programador não precisa declarar tipos para as variáveis que ele usará. Se é uma linguagem de alto nível e amiga do programador, ela deve tornar minha vida mais fácil e se preocupar com o que eu guardo nas minhas varíáveis. Ex: 

porcos = 3
galinhas = 5
vacas = 2
animais = porcos + galinhas + vacas

vacaPremiada = "Mimosa"

print("Minha fazenda possui", porcos, "porcos,", galinhas, "galinhas e", vacas, "vacas")
print("São no total", porcos + galinhas + vacas, "bichos")
print("Mas nenhum é tão querido quanto", vacaPremiada)

vacaPremiada = 200

print("Minha vaca premiada pesa", vacaPremiada, "quilos")

Tem saídas: 

Minha fazenda possui 3 porcos, 5 galinhas e 2 vacas
São no total 10 bichos
Mas nenhum é tão querido quanto Mimosa
Minha vaca premiada pesa 200 quilos

Veja que em nenhum momento precisei dizer que porcos, galinhas e vacas são do tipo inteiro. Além disso, a variável vacaPremiada era uma string no começo do programa e terminou como um valor numérico.

Variáveis em Python são completamente dinâmicas. Não somente o programador pode inicializá-las com qualquer valor, como pode modificar seu tipo durante a execução. Embora este poder esteja ao alcance, usá-lo é extremamente desaconselhável. Para que seu código tenha qualidade, é uma prática bem fundamentada não usar a mesma variável para diferentes tipo de dados. Evite as variáveis bombril, elas não tem tantas utilidades quanto você pensa.


A Prensa de van Rossum

Guido van Rossum, criador do Python

No tópico anterior vimos uma variável do tipo string, a vacaPremiada. Strings nada mais são que listas de caracteres em sequência, formando algo "imprimível". Mesmo que o dado a ser escrito seja um número ou um valor booleano (True ou False), para colocar na tela, num arquivo texto ou numa mensagem a ser transmitida por email, é necessário convertê-los para string.

A seguinte linha extraída do código acima, print("São no total", porcos + galinhas + vacas, "bichos") poderia ser trocada por: 

total = "São no total " + str(porcos + galinhas + vacas) + " bichos"
print(total)

Mas isto é menos eficiente que usar string format. Em ambos os casos vários objetos são concatenados, criando strings intermediárias. Com string format só é preciso fazer print("São no total %d bichos" % (porcos + galinhas + vacas)). Mais elegante e eficiente. Abaixo um exemplo um pouco maior de uso do string format: 

nome = 'Eduardo Tenório'
idade = 27
altura = 1.83
peso = 110 #sou denso
olhos = 'verdes'
pele = 'bronzeada de LED'
cabelo = 'ruim'

print('Vamos falar sobre %s' % nome)
print('Ele possui %1.2f de altura' % altura)
print('Pesa %d quilos (densidade alta)' % peso)
print('Possui charmosos olhos %s e pele %s' % (olhos, pele))
print('E tem cabelo %s' % cabelo)

Os números e letras depois do % ficam como exercício. Por que no segundo print existe %1.2f?


Aspas simples?

Se você prestou atenção até agora e deu uma pequena olhada nos códigos, deve ter percebido que string são englobadas por '' (aspas simples), contrariando o "padrão". Em Python strings podem ser criadas usando qualquer um dos modos, desde que o tipo de aspa que fecha seja o mesmo que abre. Logo 'olá enfermeira' e "olá enfermeira" são strings válidas. Os demais não.

Também é possível criar strings de múltiplas linhas, usando triplas de "" ou ''. A diferença é que neste caso a formatação mantém-se. Exemplos: 

stringLine = 'linha 1\nlinha 2'
stringMultLine = '''linha 1
linha 2'''

print(stringLine)
print()
print(stringMultLine)

$ python3.2 stringLine.py
linha 1
linha 2

linha 1
linha 2

E se eu quiser escrever uma aspa dentro de uma string? Como em I'm OK, por exemplo. Existem 2 maneiras: print("I'm OK") ou print('I\'m OK'). O caractere \ (barra invertida - backslash), assim como na maioria das linguagens, é utilizado como caractere de escape.
