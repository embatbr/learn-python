Dinâmico não é genérico

Para quem nunca programou numa linguagem com tipagem dinâmica, pode demorar um pouco até considerar algo agradável (ou até correto) de se fazer. Em Python, o programador não precisa declarar tipos para as variáveis que ele usará. Se é uma linguagem de alto nível e amiga do programador, ela deve tornar minha vida mais fácil e se preocupar com o que eu guardo nas minhas caixas. Ex: 

porcos = 3
galinhas = 5
vacas = 2
animais = porcos + galinhas + vacas

vacaPremiada = "Mimosa"

print("Minha fazenda possui", porcos, "porcos,", galinhas, "galinhas e", vacas, "vacas")
print("São no total", porcos + galinhas + vacas, "bichos")
print("Mas nenhum é tão querido quanto", vacaPremiada)

vacaPremiada = 200

print("Minha vaca premiada pesa", vacaPremiada, "quilos")

Tem saídas: 

Minha fazenda possui 3 porcos, 5 galinhas e 2 vacas
São no total 10 bichos
Mas nenhum é tão querido quanto Mimosa
Minha vaca premiada pesa 200 quilos

Veja que em nenhum momento precisei dizer que porcos, galinhas e vacas são do tipo inteiro. Além disso, a variável vacaPremiada era uma string no começo do programa e terminou como um valor numérico.

Variáveis em Python são completamente dinâmicas. Não somente o programador pode inicializá-las com qualquer valor, como pode modificar seu tipo durante a execução. Embora este poder esteja ao alcance, usá-lo é extremamente desaconselhável. Para que seu código tenha qualidade, é uma prática bem fundamentada não usar a mesma variável para diferentes tipo de dados. Evite as variáveis bombril, elas não tem tantas utilidades quanto você pensa.
